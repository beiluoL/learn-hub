## 常见排序算法与快速排序

**核心结论**：快排是实际应用中最快的通用排序算法——均摊 O(nlogn)，原地排序，缓存友好。虽然最坏情况退化为 O(n²)，但通过三数取中或随机 pivot 可有效规避。

---

### 常见排序算法一览

**一句话描述**：

- **冒泡排序**：相邻元素两两比较，大数逐渐"浮"到末尾，循环 n-1 轮。
- **选择排序**：每轮从未排序区间找到最小值，放到已排序区间的末尾。
- **插入排序**：将每个元素插入到已排序部分的正确位置，适合小数据量或基本有序的数据。
- **希尔排序**：插入排序的改进版，使用递减间隔进行多次插入排序，最终间隔为 1。
- **归并排序**：分治——递归将数组分成两半排序，再合并两个有序数组，稳定但不原地。
- **堆排序**：将数组构建成最大堆，反复取出堆顶（最大值）放到末尾，调整堆。
- **快速排序**：选 pivot，分区（小的在左、大的在右），递归排序左右部分。

### 快速排序原理

快排基于**分治思想**，核心是分区（Partition）操作：

```python
def quicksort(arr, left, right):
    if left >= right:
        return
    pivot_idx = partition(arr, left, right)  # 分区，返回 pivot 最终位置
    quicksort(arr, left, pivot_idx - 1)       # 递归左区间
    quicksort(arr, pivot_idx + 1, right)      # 递归右区间

def partition(arr, left, right):
    pivot = arr[right]          # 选最右元素为 pivot
    i = left                    # i 指向"小于区间"的末尾
    for j in range(left, right):
        if arr[j] < pivot:
            arr[i], arr[j] = arr[j], arr[i]
            i += 1
    arr[i], arr[right] = arr[right], arr[i]  # 将 pivot 放到正确位置
    return i
```

**执行过程**（以数组 [3, 7, 8, 5, 2, 1, 9, 5, 4]，pivot=4 为例）：

```
初始：      [3, 7, 8, 5, 2, 1, 9, 5, |4]    pivot=4
j=0,3<4：   [3, 7, 8, 5, 2, 1, 9, 5,  4]    i=1（3 在正确位置）
j=1,7>4：   [3, 7, 8, 5, 2, 1, 9, 5,  4]    i=1（不动，7 跳过）
j=2,8>4：   [3, 7, 8, 5, 2, 1, 9, 5,  4]    i=1
j=3,5>4：   [3, 7, 8, 5, 2, 1, 9, 5,  4]    i=1
j=4,2<4：   [3, 2, 8, 5, 7, 1, 9, 5,  4]    i=2（2 与 7 交换）
j=5,1<4：   [3, 2, 1, 5, 7, 8, 9, 5,  4]    i=3（1 与 8 交换）
j=6,9>4：   [3, 2, 1, 5, 7, 8, 9, 5,  4]    i=3
j=7,5>4：   [3, 2, 1, 5, 7, 8, 9, 5,  4]    i=3
最终：      [3, 2, 1, 4, 7, 8, 9, 5,  5]    pivot=4 归位
              <4的区域           >4的区域
```

**时间复杂度分析**：

- **最好/平均**：每次分区 pivot 刚好将数组分成大致相等的两半。递归树深度 logn，每层扫描 n 个元素——O(nlogn)。
- **最坏（倒序或正序）**：pivot 始终是最值，每次分区一边为 n-1 个元素，一边为空。递归退化为链表——O(n²)。

**避免最坏情况**：三数取中法（从 arr[left]、arr[mid]、arr[right] 中选择中间值作为 pivot）或随机选择 pivot，可将最坏概率降到极低。

### 为什么实际中快排比归并和堆排更快

虽然理论时间复杂度同为 O(nlogn)，但快排在实际基准测试中通常明显快于归并和堆排。

**三个核心原因**：

1. **缓存友好（Cache Locality）**：快排的数据访问模式是按顺序扫描数组的局部区间（partition 是连续扫描），充分利用 CPU 缓存行。归并排序需要 O(n) 额外空间且在合并阶段跨区域读写，堆排序的父子节点在数组中物理位置不连续（索引跳跃式访问），缓存未命中率高。

2. **常数因子小**：快排的比较和交换都在寄存器或 L1 缓存内完成。一次交换操作通常对应 2-3 条 CPU 指令。堆排序每次"上浮/下沉"都需要多次比较，且元素间位置不连续导致大量内存访问。

3. **原地排序，空间开销仅 O(logn)**：递归调用的栈深度。归并排序需要 O(n) 辅助数组，大量内存分配和拷贝导致实际运行变慢。

**何时用归并**：需要稳定排序（保留相等元素的相对顺序）、处理链表（链表天然适合归并，不浪费辅助空间）、外部排序（数据量大到无法全部载入内存）。

**何时用堆排**：需要同时选 Top K（堆天然支持）、对最坏情况敏感的场景、极度内存受限。

### 面试官追问

**追问**：快排稳定吗？如何实现稳定快排？

**回答**：标准快排不稳定——分区过程中相等元素的相对位置可能改变（跳跃式交换）。可以在分区时使用辅助数组记录原始索引，但会牺牲原地性和部分性能。Java 中 Arrays.sort() 对对象类型使用 TimSort（归并的改进版，稳定），对基本类型使用 Dual-Pivot Quicksort（双轴快排，不够稳定但基本类型不关心稳定性）。

**追问**：什么是三路快排？

**回答**：普通快排对大量重复元素效率低——等于 pivot 的元素仍参与递归。三路快排将数组分为 <pivot、=pivot、>pivot 三个区域，等于 pivot 的区域不再递归。对大量重复元素的排序（如按颜色排列国旗），三路快排接近 O(n)。荷兰国旗问题是其经典应用。

**追问**：外部排序怎么做？

**回答**：数据量超过内存时，用归并排序思想。第一步：将大文件切分为多个内存能装下的小块，每块内部排序后写回磁盘。第二步：多路归并——打开 K 个有序文件，用最小堆每次取出当前最小值（K 个文件的当前头部中最小者），输出到最终文件。堆大小等于 K，时间复杂度 O(nlogK)，IO 为 O(n) 次磁盘读写。
